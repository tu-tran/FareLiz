FareLiz - Flight Fare Scanner

FareLiz is a flight fare scanner implemented using .NET (C#)

The code is licensed under the MIT license. See opensource.org and tl;dr for more details.

Features

  • Scan for flight fare for certain routes in multiple time period
  • Real-time monitoring and notify users as soon as the price changes. It also support highly configurable settings to get the most desirable flight fare (e.g. minimum price difference before triggering the notification, price limit...)
  • Command line interface for automated tasks
  • Automatic fare scanning using scheduled tasks in Windows
  • Fare data storage and analysis
  • Cloud storage synchronization: Users can synchronize data with cloud

Getting started

For end-users

  1. Go to Release page to download the binaries (in ZIP or installation package for x86 and x64 system)
  2. Extract/Install the binaries
  3. Start FareLiz. At first run, the application will ask you to select the appropriate plugins and their configurations.
  4. The main entry point will be the flight statistics view (which contains historical fare data). From here, the users can initiate different functionalities:
  5. Scan/store new fare data, click on Fare Scanner
  6. Schedule an automated fare scanning, click on Scheduler
  7. Change the software configurations, click on Settings

For developers

The application also provides support for the implementation of 3rd-party plugins. This allows the developers to extend the application to support additional:

  • Travel agencies data source
  • Fare data storage engine
  • Cloud data storage synchronization All plugin assemblies will follow the name pattern *.Plugins.dll and should be located in the same path as the main executable. To get started, the main solution file is FareLiz.sln. Example implementations of the plugins can be found in Plugins folders of the solution.

History

The project was implemented for personal use: it has been difficult to track down the airline ticket pricing mechanisms, and the existing services were not powerful enough: lack of data/lots of email spam/PAID services. Thus the project was initiated as a one-man project for the sake of simplifying the fare hunting process. As a frequent traveler, it is always easier to pick the annual travel time based on the the price chart from the previous years --> FareLiz was developed.
